2  Data

2.1 Description

As we are studying crimes at NYC parks and potential reasons, we collected three datasets: crimes, unemployment, and mobility data.

First, we collected crime data in NYC from NYPD official websites focused on crime statistics. The link of data is NYPD Crime Statistics. The data covers quaters from 2014 to 2023. Each quater dataset shows features like numbers of crimes, park sizes, and boroughs. For our study, we focused on three factors: total number of crimes, park size categorites, and boroughs.

Second, we collected unemployment data from U.S. Bureau of Labor Statistics official website focused on NYC area. The link of data is NYC Unemployment Statistics. Additionally, BLS Data Finder offered in this website is able to search unemployment data for NYC. Monthly unemployment data is available from 2013 to 2023.

Last, we collected mobility data from Google Community Mobility Reports. The link of data is Mobility Data. Daily data of movement trend in places such as parks is available in the Covid-19 period.

2.2 Research Plan

  1. Time series analysis for crimes data. We conducted time series analysis to look into components including long term trends, seasonality (systematic, calendar related movements), and irregularity (unsystematic, short term fluctuations).

  2. Heatmaps and bar charts for crimes data. We illustrated crimes data in heatmaps based on boroughs and periods to find time and regional differences. Bar charts are used to show the specific numbers of crimes for boroughs.

  3. Relationship between crimes and employment rate. We looked into trends for total number of crimes and unemployment rates both anually and monthly. We first demonstrated these two variables in scatterplots.

  4. Relationship between crimes and mobility data. We looked into the crimes and mobility data for different parks and boroughs, investigating whether visitors flow affects crimes.

It’s obvious that the missing values are average unemployment rates for each year and the last two months in 2023. For the average rates, we can direcly compute the average rates based on monthly rates.

Code 
unemployment_rate$Avg <- rowMeans(unemployment_rate[, 2:13], na.rm = TRUE)

For the last two months in 2023, it’s reasonable because it does not have any historical data. We use the data for Oct to fill in missing value for Nov and Dec.

Code 
last_row <- nrow(unemployment_rate)

unemployment_rate[last_row, "Nov"] <- unemployment_rate[last_row, "Oct"]
unemployment_rate[last_row, "Dec"] <- unemployment_rate[last_row, "Oct"]

# re-compute avarage unemployment rate
unemployment_rate[last_row, "Avg"] <- rowMeans(unemployment_rate[last_row, 2:13], na.rm = TRUE)
